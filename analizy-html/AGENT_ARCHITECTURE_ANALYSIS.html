Wzorce Orkiestracji 2026

Prosty ale kruchy

Flat Routing

Klasyfikator → jeden specjalista. Najniższa latencja i koszt. Nie radzi sobie z multi-intent.

User → Router → Agent A/B/C
Przewidywalny ale wolny

Sequential Pipeline

Agenty w stałej kolejności. Każdy output = input następnego. Kumuluje latencję (6-9s na 3 agenty).

A → B → C → Output
REKOMENDOWANY — Produkcyjny standard

Hierarchical Orchestration

Centralny orkiestrator dekomponuje zadania, deleguje do specjalistów (równolegle/sekwencyjnie), syntetyzuje wyniki. Graceful degradation.

         Orchestrator
        /     |      \
    Agent_A  Agent_B  Agent_C
        \     |      /
         Synthesis
70-90% redukcja kosztów

Plan-and-Execute

Drogi model planuje (Opus/Sonnet), tanie modele wykonują (Haiku). 100K dziennych interakcji: $900 → $150.

Opus (plan) → Haiku (execute×N) → Sonnet (synthesize)

Anti-patterny i Błędy

Matematyka nie kłamie: jeśli każdy krok ma 85% szans na sukces, to po 5 krokach masz 44%, a po 10 krokach tylko 20%.

🔄

Infinite Loop

Agenty w pętli krytyki bez limitu iteracji. Fix: max 3 cykle.

🌊

Hallucination Cascade

Agent A halucynuje → Agent B buduje na fałszywych danych. Fix: source tracking.

💸

Token Waste

15 agentów na problem wymagający 3. Fix: zacznij od 3, dodawaj gdy konieczne.

🧩

Over-Engineering

800-liniowy katalog zamiast 80 linii i URUCHOM. Fix: start simple.

📉

Context Pollution

Za dużo kontekstu = gorsza jakość. Fix: narrow context per agent.

🔗

Cascading Failures

Jeden agent pada → wszystko pada. Fix: circuit breaker pattern.

⏱️

Coordination Overhead

Więcej agentów = więcej overhead. Diminishing returns po 11+.

🎭

Sycophancy Chain

Agenty zgadzają się ze sobą zamiast kwestionować. Fix: Devil's Advocate.

📞

Schema Drift

Dane mutują między agentami (gra w głuchy telefon). Fix: JSON Schema validation.

Poziom: INTERMEDIATE+ — doskonałe instynkty, luka między wizją a implementacją

Gold Standard Architecture 2026

Idealny szablon — 14 agentów, 5 faz

     A-00 Orchestrator (state manager)
     A-01 Synthesizer (Wolny Elektron 2.0)
     ┌─────────────────────────────────┐
     │  PHASE 1: RESEARCH (parallel)   │
     │  A-02 Researcher (web)          │
     │  A-03 Researcher (codebase)     │
     │  A-04 Researcher (domain)       │
     │  A-05 Research Critic           │  ← NEW!
     └──────────────┬──────────────────┘
                    ↓ GATE: Synthesis Report
     ┌─────────────────────────────────┐
     │  PHASE 2: BUILD (sequential)    │
     │  A-06 Backend Builder           │
     │  A-07 Frontend Builder          │
     │  A-08 Feature Builder           │
     │  A-09 Build Critic              │  ← NEW!
     └──────────────┬──────────────────┘
                    ↓ GATE: All reviewed
     ┌─────────────────────────────────┐
     │  PHASE 3: QA (parallel)         │
     │  A-10 Security Auditor          │
     │  A-11 Functional Tester         │
     └──────────────┬──────────────────┘
                    ↓ GATE: All PASS
     ┌─────────────────────────────────┐
     │  PHASE 4: FINALIZE              │
     │  A-12 Code Packager             │
     │  A-13 Documentation Writer      │
     └─────────────────────────────────┘

     Model Routing:
     Opus  → A-00, A-01 (orchestrator + synthesizer)
     Sonnet → A-06-A-09, A-12-A-13 (builders + finalization)
     Haiku  → A-02-A-05, A-10-A-11 (research + QA)

     Estimated cost: ~$25 (medium project)

Kluczowe ulepszenia vs Twoje systemy: Research Critic PRZED syntezą, Build Critic na każdym milestone, progressive quality gates zamiast end-phase QA.
