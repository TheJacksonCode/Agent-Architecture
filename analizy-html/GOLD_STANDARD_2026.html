Section 01

Core Principles

Five inviolable rules governing every decision in the architecture.

Principle 01
Composite Pattern Architecture
Hub-and-Spoke for orchestration, Handoff for phase transitions, Mesh for QA. Never use a single pure pattern.
Principle 02
Narrow Context, Deep Specialization
Every worker agent gets ONLY the context it needs. The lower in the hierarchy, the narrower the context window.
Principle 03
Deterministic Flow + Escape Hatches
Strict phase protocol with well-defined rollback paths when things fail. Adaptive, not rigid.
Principle 04
Manifest as Single Source of Truth
All inter-agent communication goes through a central, machine-readable Manifest. No ambient context.
Principle 05
Human-in-the-Loop at Gate Checks
The user approves phase transitions, not individual agent outputs. Strategic oversight, not micromanagement.

Section 04

Phase Protocol

5-phase protocol: Initialization, Research, Implementation, QA, and Finalization. Each with strict gate criteria.

Phase 0
Initialization
Mode Sequential
Agents A-00 only
Output MANIFEST.json, AGENTS-CATALOG.md
1. A-00 analyzes user requirements 2. A-00 audits existing project structure 3. A-00 creates MANIFEST.json with phase definitions 4. A-00 creates/updates AGENTS-CATALOG.md 5. A-00 presents plan to user for approval
Gate 0 -- User approves plan (Required)
Phase 1
Research
Mode Parallel (Hub-and-Spoke)
Agents A-02, A-03, A-04, A-05, A-01
Output Research reports, Synthesis Report
1. A-02, A-03, A-04 execute in PARALLEL A-01 monitors and asks clarifying questions Each produces structured JSON report 2. A-05 critiques each report ACCEPT -> proceed | REVISE -> re-do (max 2 iterations) 3. A-01 synthesizes all accepted reports into Synthesis Report 4. A-00 validates Synthesis Report completeness Updates MANIFEST.json
Gate 1 -- Critic + Orchestrator approve (User optional)
Phase 2-3
Implementation
Mode Handoff + Peer Review
Agents A-06, A-07, A-08, A-09, A-01
Output Working code with tests
1. A-06 implements core/backend (milestone 1) -> A-07 reviews, A-08 reviews (cross-review) -> A-09 critiques milestone 1 -> IF REVISE: A-06 fixes (max 3 iterations) 2. A-07 implements frontend/UI (milestone 2) -> A-08 reviews, A-06 reviews (cross-review) -> A-09 critiques milestone 2 3. A-08 implements features (milestone 3) -> A-06 reviews, A-07 reviews (cross-review) -> A-09 critiques milestone 3 4. Integration pass (all builders verify interop)
Gate 2 -- All milestones accepted, tests pass
Phase 4
Quality Assurance
Mode Mesh (Parallel QA)
Agents A-10, A-11, A-12
Output Audit reports
1. A-10, A-11, A-12 execute in PARALLEL Each produces independent report 2. A-00 aggregates findings CRITICAL -> STOP, loop back to Phase 2-3 HIGH -> loop back to responsible builder MEDIUM/LOW -> add to backlog 3. IF any loop-backs: re-run affected QA agent after fix
Gate 3 -- Clean QA, user approves (Required)
Phase 5
Finalization
Mode Sequential
Agents A-13
Output Production-ready package
1. A-13 cleans up code, generates documentation 2. A-00 presents final package to user
Gate 4 -- User accepts final delivery (Required)

Key Innovations

What Makes Gold Standard Different

01
Wolny Elektron 2.0
Textinio's best idea, improved with bidirectional communication, persistent availability, and structured chain-of-thought logging.
02
Progressive Quality Gates
Instead of QA only at the end (all three architectures), critics validate at EVERY phase. The single biggest improvement for reducing rework.
03
Dedicated Research Critic (A-05)
None of the source architectures validated research before synthesis. This prevents garbage-in-garbage-out.
04
Token Budget Awareness
None of the source architectures tracked or limited token usage. Gold Standard includes per-agent budgets and cost estimation.
05
Model Routing
Opus for complex tasks (orchestration, synthesis), Sonnet for workers. Saves 3-5x on cost without quality loss for worker tasks.
06
Merged Finalization
Textinio used two agents (F-01 + F-02). Gold Standard merges these into A-13, reducing overhead without losing capability.
07
Five Minds as Critic
CV Project's Five Minds debate was creative but token-expensive. The Critic pattern achieves the same quality assurance through structured rubric evaluation.
08
Formal Error Classification
Textinio had basic recovery loops. Gold Standard classifies errors by type and routes them to the correct recovery path.
